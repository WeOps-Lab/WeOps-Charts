# node-exporter pprof 漏洞修复方案

## 漏洞说明
- **工单编号**: SINC20260126000005
- **漏洞**: Golang pprof 信息泄露（端口 9100 暴露 `/debug/pprof/*` 路径）
- **修复方式**: 添加 nginx sidecar 容器过滤 pprof 请求，仅允许访问 `/metrics`

---

## 部署步骤

### 1. 替换文件
```bash
cd weops-kubernetes-collect/charts/
cp -r node-exporter node-exporter.backup     # 备份
rm -rf node-exporter
unzip node-exporter.zip                       # 解压新文件
```

### 2. 执行部署
```bash
cd weops-kubernetes-collect/
helm upgrade weops-collecter . -n weops
kubectl get pod -n weops1 -l app=node-exporter    # 等待 Pod 变为 2/2 Running
```

---

## 验证修复

```bash
# 一键验证所有节点
for pod in $(kubectl get pod -n weops -l app=node-exporter -o name); do
  NODE_IP=$(kubectl get -n weops $pod -o jsonpath='{.status.hostIP}')
  echo "节点 $NODE_IP:"
  curl -s -o /dev/null -w "  pprof: %{http_code}, metrics: " http://$NODE_IP:9100/debug/pprof/
  curl -s -o /dev/null -w "%{http_code}\n" http://$NODE_IP:9100/metrics
done
```

**预期结果**：`pprof: 403, metrics: 200`

---

## 回退方法

```bash
# 方式1：Helm 回滚（推荐）
helm rollback weops-collecter -n weops

# 方式2：还原文件后重新部署
cd weops-kubernetes-collect/charts/
rm -rf node-exporter
mv node-exporter.backup node-exporter
cd ..
helm upgrade weops-collecter . -n weops
```
